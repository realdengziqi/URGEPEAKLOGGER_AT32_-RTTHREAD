# 罗氏线圈参数确认清单

本文用于向客户或传感器供应商确认外部浪涌传感器参数。当前客户只提供了几个线圈，并说明为“罗氏线圈”。目前已确认外部罗氏线圈没有积分器，但尚未提供线圈互感、比例系数、频响、触发阈值、峰值保持窗口等关键参数。未完成标定前，固件只能可靠记录 `raw ADC`、触发状态和相对峰值，不能宣称已经得到真实 kA/kV 峰值。

## 竞品手册依据

参考资料：`competing_product_info/浪涌峰值记录终端（三种传感器）-用户手册 - 中性.doc`。

手册中可提取到以下与传感器相关的信息：

- 记录内容包括浪涌次数、极性、峰值、波前时间、半峰值时间、Q 值、能量、电流陡度、发生日期和时间，最大计数 9999 次。
- 触发电流为 `0.5 / 1 / 2.5 / 5 kA` 可选，默认 `5 kA`，分辨率 `0.1 kA`。
- T1 使用 `10/350us` 线圈，T2/T3 使用 `8/20us` 线圈。
- 出厂默认配置 `8/20us` 线圈，如需 `10/350us` 线圈需要订货说明。
- 线圈类型包括：
  - `8/20us` 波形罗氏线圈：刚性闭口金属外壳。
  - `10/350us` 波形罗氏线圈：刚性闭口塑胶外壳。
  - `10/350us` 波形罗氏线圈：柔性开口，采集范围 `0.5~100 kA`。
- 传感器线标配 `1 m`，长度可定制；电缆超过 `5 m` 或监测 T1 类型浪涌时，传感器需要单独调校。
- 传感器接口为 `SMA`。
- 多脉冲记录可识别最多 `20` 个连续脉冲，竞品资料中提到脉冲间隔 `>=40 ms`。
- 安装上要求雷击传感器与防雷接地隔离，避免地电位反击损坏仪表。

由此可以判断：浪涌传感器不是只确认一个比例系数就够了。不同波形、不同线圈结构、不同线缆长度、是否带积分器，都会影响触发阈值、峰值换算、波前/半峰值计算、能量计算和多脉冲识别。

## 已知线索：1KA/10mV

用户观察到竞品线圈名称或标识上有 `1KA/10mV`。初步推断，这很可能表示线圈或传感器灵敏度为：

```text
1 kA -> 10 mV
即 10 mV/kA
```

这个标识非常有价值，但仍不能单独作为最终固件换算依据，原因如下：

- 外部线圈已确认没有积分器，所以需要确认该标识对应的是原始线圈在什么波形、什么负载下的输出。
- 需要确认 `10 mV/kA` 是针对 `8/20us`、`10/350us`，还是某个指定标准波形。
- 需要确认标定时的线缆长度、终端负载、输出阻抗和示波器输入阻抗。
- 需要确认输出是否双极性。如果正负浪涌都会出现，ADC 前端必须有偏置或双极性处理方案。
- 需要确认该标识的大小写和完整型号，`KA` 应理解为 `kA`，但仍建议拍照留档。

如果按 `10 mV/kA`、`3.3 V` 参考电压、12 位 ADC 粗略估算：

```text
1 LSB = 3.3 V / 4096 = 0.805 mV
1 kA = 10 mV = 约 12.4 ADC count
0.5 kA = 5 mV = 约 6.2 ADC count
100 kA = 1000 mV = 约 1241 ADC count
```

这说明如果传感器直接接入 MCU ADC，低阈值触发会非常依赖前端噪声、ADC 偏置稳定性和滤波策略。若产品要支持 `0.5 kA` 或 `0.1 kA` 分辨率，通常需要合适的模拟增益、偏置、比较器或高速采样方案，而不能只靠每秒低频 raw ADC 读取。

## 本板浪涌模拟前端初步分析

依据：

- BOM：`hardware_info/主控板_BOM.csv`
- 网表：`hardware_info/主控板_网表.csv`

相关器件和网络：

- `RF1`：SMA 输入连接器，中心针接 `$7N3`，外壳接 `AGND`。
- `U20`：`TLV2372IDR` 双运放，BOM 标称 GBP `3MHz`、压摆率 `2.1V/us`。
- `U5`：`TP7660` 电荷泵，配合 `OP_+5V / OP_-5V` 给运放提供正负电源。
- `U19 pin 9`：`MCU_PC1_ADC2_IN11`，当前作为浪涌 ADC 输入。

从网表看，当前模拟链路大致为：

```text
SMA center RF1 pin5 ($7N3)
  -> R41 22k
  -> $7N4
  -> R43 3.4k 到 AGND
  -> C51 33pF 到 AGND
  -> D3 到 OP_+5V / OP_-5V 钳位
  -> U20A 同相跟随器
  -> R42 1k
  -> $7N13
  -> R38 1k 到 3V3A
  -> U20B 同相跟随器
  -> R39 51R
  -> C50 1nF 到 AGND
  -> R40 0R
  -> MCU_PC1_ADC2_IN11
```

按这个拓扑推断：

- `U20A` 是第一路缓冲，`U20 pin1/pin2` 同网，`pin3` 接输入分压节点。
- `U20B` 是第二路缓冲，`U20 pin6/pin7` 同网，`pin5` 接偏置节点。
- `R41=22k` 与 `R43=3.4k` 构成输入衰减，比例约为 `3.4 / (22 + 3.4) = 0.134`。
- `R42=1k` 与 `R38=1k` 把第一路输出和 `3V3A` 做平均，形成约 `1.65V` 的 ADC 中点偏置，同时信号幅度再乘 `0.5`。
- 因此 SMA 中心到 ADC 的交流小信号增益约为 `0.134 * 0.5 = 0.067`。
- `R39=51R` 与 `C50=1nF` 是 ADC 前的低通/隔离，时间常数约 `51ns`，截止频率约 `3.1MHz`。

这个前端不像积分器。它没有看到典型的 `输入电阻 + 反馈电容` 运放积分结构，也没有看到用于低频泄放的反馈电阻并联电容。更像是：

```text
输入保护 + 衰减 + 双电源缓冲 + 1.65V 偏置 + ADC RC 滤波
```

外部传感器已确认没有积分器，因此本板当前前端大概率没有完成电流积分，ADC 采到的仍然更接近 `di/dt` 相关波形，只是被衰减、偏置并缓冲后送入 ADC。这样不能直接用峰值 raw 表示电流峰值 kA。

如果 `1 kA -> 10 mV` 是线圈在 SMA 端的输出灵敏度，则经过本板前端后，ADC 端幅度约为：

```text
ADC signal ~= 10 mV/kA * 0.067 = 0.67 mV/kA
1 LSB ~= 0.805 mV
所以约为 0.83 ADC count/kA
```

这个量级太小，不足以支撑 `0.5 kA` 触发或 `0.1 kA` 分辨率。因此有两种可能需要进一步确认：

- `1KA/10mV` 不是 SMA 端真实输出灵敏度，或者不是我们当前线圈的参数。
- 当前硬件前端原本面向更高幅度的原始罗氏线圈脉冲，不是面向 `10mV/kA` 的已积分小信号输出。

结论：外部线圈已确认无积分器，且仅凭现有网表和 BOM 不能认为板上运放已经完成罗氏线圈积分。因此后续真实浪涌测量路线应确定为“高速 ADC 采样 + 数字积分”。线圈参数暂时未知不阻塞采样框架设计，但会影响最终 kA 换算、触发阈值和精度声明。后续仍需要在示波器上同时观察 `RF1 SMA 中心`、`U20A 输出`、`U20B 输出/PC1 ADC`，并用已知浪涌或信号源验证整体传递函数。

## 原始罗氏线圈和带积分器传感器的区别

当前外部传感器已确认为原始罗氏线圈，线圈输出近似满足：

```text
v_out = M * di/dt
```

其中 `M` 是互感系数。此时 ADC 看到的是电流变化率相关信号，不是电流本身。要得到真实电流峰值，必须具备以下条件之一：

- 模拟前端包含积分器，输出已经变成与电流成比例的电压。
- 固件以足够高采样率采集原始波形，并做数字积分，同时已知线圈互感、前端增益、偏置、采样率和积分初值处理策略。

带积分器传感器的输出通常可按：

```text
current_kA = output_voltage / sensitivity
```

这种情况下固件可以把触发阈值配置为 kA，再根据比例系数、ADC 参考电压、前端增益和偏置换算成 raw threshold。但本项目当前外部线圈已确认没有积分器，所以不能按这个路径设计第一版真实浪涌采集。

因此，当前路线应直接按原始罗氏线圈处理：PC1 的 ADC 峰值 raw 不能直接换算为真实浪涌电流峰值，必须高速采样并做数字积分。

## 对固件参数的影响

### 触发阈值

正式产品中，用户或协议侧应配置 `kA` 阈值，例如 `0.5~100 kA` 范围内按 `0.1 kA` 步进。固件内部再换算为 ADC raw 阈值。

在传感器参数未确认前，只能保留当前的 `threshold_delta_raw` 调试方式，不应把它解释为真实 kA。

### 峰值保持窗口

峰值保持窗口必须覆盖完整浪涌波形和前端恢复时间：

- `8/20us` 波形：建议窗口至少大于 `100 us`，实际要结合采样率和前端带宽确定。
- `10/350us` 波形：建议窗口至少大于 `1~2 ms`，实际要结合线圈/积分器响应确定。
- 如果要记录半峰值时间、能量或 Q 值，不能只保持峰值，还需要保存一段波形或提取足够多的时间特征点。

### 采样率和触发方式

当前低频 `data_service` 每秒读取 PC1 raw 只适合后端通路验证，不适合真实浪涌采集。既然外部线圈已确认无积分器，真实采集必须采用高速 ADC 采样并进行数字积分。

后续真实采集至少需要评估：

- 定时器触发 ADC，配合 DMA 环形缓冲。
- 模拟比较器或高速外部中断用于预触发。
- ADC 采样率是否足以捕获 `8/20us` 上升沿。
- 预触发缓冲是否需要保存触发前若干采样点，用于判断极性和基线。
- 前端是否需要双极性输入偏置，因为罗氏线圈可能输出正负脉冲。
- 数字积分的基线扣除、积分泄放、窗口截断和饱和恢复策略。

### 记录字段

如果要靠近竞品能力，后端记录至少要预留：

- 触发时间。
- 峰值电流 kA。
- 极性。
- 波前时间。
- 半峰值时间。
- 电流陡度。
- 能量或近似能量。
- 多脉冲序号、间隔时间和每个脉冲峰值。

但这些字段是否能真实计算，取决于采样方案、线圈参数和前端带宽，不能先在 UI 或 Modbus 上承诺。

## 样品测试建议

如果客户无法提供完整资料，至少需要配合做一次受控标定：

1. 使用已知峰值和标准波形的浪涌电流源，例如 `8/20us 1 kA / 5 kA / 10 kA` 或 `10/350us` 对应测试点。
2. 同时用示波器采集传感器输出和仪表 ADC 前端输入，记录峰值、电压范围、极性、上升沿、半峰值时间和恢复时间。
3. 分别测试当前线缆长度和计划量产线缆长度。
4. 如果有多种线圈，每种线圈单独建立参数，不要共用一个比例系数。
5. 若现场只允许低能量测试，也应至少获得供应商标定报告，并在固件中把参数标记为“待实浪涌校准确认”。

## 当前固件处理原则

- 未拿到线圈参数前，`surge_peak_value_milli_kv` 或后续 `surge_peak_value_milli_ka` 不应输出假换算值。
- Modbus/UI/MSH 可以展示 raw、delta raw、触发计数和状态机，但必须标注为调试量。
- 真实浪涌采集方案确定前，不应把每秒低频 ADC 采样当作产品级浪涌捕获方案。
- 真实浪涌采集路线已确定为高速 ADC 采样 + 数字积分；线圈参数未确认前，先实现 raw 波形捕获、基线、触发、峰值和积分框架。
- 后端可以先保留 `surge_service` 状态机和缓存接口，后续替换底层高速采样实现，不影响 Modbus 响应及时性原则。
