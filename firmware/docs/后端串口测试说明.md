# 后端数据服务串口测试说明

当前阶段以建立设备内部数据后端为主，UI 自动初始化和按键翻页轮询已经暂停。后端验证优先通过 COM12 的 RT-Thread MSH 命令完成。

## 模块边界

- `data_service`：维护统一数据快照，当前包含 RTC、HDC1080 环境温湿度、NTC、DI、浪涌 raw/峰值状态和运行期参数状态。
- `surge_service`：维护 PC1 浪涌输入的 raw 状态、armed/latched 状态、峰值 raw、触发计数和注入测试入口；真实 kV 换算待外部传感器确认后接入。
- `param_service`：维护运行期参数表，支持范围检查，并已接入 AT24C128 EEPROM KVDB 保存/加载。
- `record_service`：维护运行期事件记录环形缓冲，当前容量 16 条；已接入片内 Flash 的 FlashDB TSDB，并通过 `record_wr` 线程异步持久化。
- `app_shell`：只作为开发阶段串口测试前端，通过后端服务接口读写数据，不直接改业务结构体。

## 推荐测试顺序

1. 查看后端总览：

   ```text
   backend
   ```

2. 查看实时快照：

   ```text
   data
   data watch 5 1000
   ```

3. 查看硬件采集 raw 值：

   ```text
   di
   di watch 10 200
   adc
   adc watch 20 200
   ```

   期望：`di` 输出 TI1..TI5 的底层 raw bit；底层 bit0..bit4 对应业务 TI1..TI5，也可称为 DI0..DI4。`adc` 输出 `PC0/ADC1_IN10` 的 NTC raw 值、换算温度和 `PC1/ADC2_IN11` 的浪涌输入 raw 值。

   当前 NTC 使用 10k 热敏电阻、10k 分压电阻、B=3380 参数，按 `3.3V -> 10k 固定电阻 -> ADC 节点 -> NTC -> GND` 拓扑换算。`data_service` 每 1 秒低频读取一次 ADC raw，Modbus 只读取已缓存快照，不在回调中触发 ADC 采样。

4. 测试浪涌后端状态机：

   ```text
   surge
   surge threshold 300
   surge arm
   surge inject 2500
   surge inject 1600
   surge clear
   ```

   期望：`surge arm` 以当前 PC1 raw 作为 baseline；`surge inject` 可在没有外部浪涌传感器时模拟 raw 跳变，超过 `threshold` 后进入 `latched` 状态并增加触发计数。

5. 测试参数读写和持久化：

   ```text
   param list
   param set modbus_addr 2
   param save
   param set modbus_addr 3
   param load
   param get modbus_addr
   param status
   ```

   期望：`param load` 后 `modbus_addr` 恢复为保存时的值 `2`，`dirty=0`，`persistent_ready=1`。

6. 测试记录服务：

   ```text
   record status
   record add 31500
   record latest
   record list
   record clear
   ```

   期望：`record add` 从 `data_service` 快照生成一条手动测试记录，并使用命令行给定的峰值；`record status` 中 `persist pending` 最终回到 0，`failed` 保持 0。

7. 测试 RTC 读写：

   ```text
   rtc
   rtc set 2026-06-21 19:30:00
   rtc
   ```

## 当前待确认

- PC0 / ADC1_IN10 已确认为 NTC 输入，PC1 / ADC2_IN11 已确认为浪涌输入；NTC 已按 10k/B3380/10k 分压接入第一版换算，分压拓扑仍建议结合原理图最终确认。
- 浪涌 raw 峰值和状态机已接入，真实峰值算法、高速采样触发方案和 kV 换算系数仍待确认，当前 `surge_peak_value_milli_kv` 保持为 0。
- 记录持久化区域已按 `存储布局设计.md` 接入片内 Flash TSDB；记录格式、历史分页查询接口、擦写寿命验收策略仍需继续完善。
- 参数服务当前使用 FlashDB KVDB 的 `param_image` 保存参数镜像；正式分区见 `存储布局设计.md`。

| 15 | surge_status，0 idle，1 armed，2 latched |
