# 非浪涌报警记录格式

本文先固定 NTC 和 TI 开关量输入两类非浪涌报警的数据格式。浪涌详情记录单独设计；浪涌高阈值报警可以复用本文的报警事件日志模型。

## 目标

非浪涌报警需要支持两类数据：

- 当前活跃报警：用于 UI、Modbus、MSH 查看设备当前是否仍处于报警状态。
- 历史报警事件记录：用于持久化保存报警发生、报警恢复和瞬时报警事件，便于后续查询。
- 活跃报警恢复镜像：用于设备掉电或复位后恢复尚未闭合的当前报警。

## 历史报警事件记录

报警历史按事件日志保存，不再把一次报警从发生到恢复合并成一条闭合记录。

支持的事件动作：

| 动作 | 含义 | 适用对象 |
| --- | --- | --- |
| `alarm_start` | 报警发生 | NTC、TI |
| `alarm_clear` | 报警恢复 | NTC、TI |
| `alarm_instant` | 瞬时报警事件 | 浪涌高阈值报警 |

每条历史报警事件记录格式固定为：

| 字段 | 含义 | 说明 |
| --- | --- | --- |
| `sequence` | 记录序号 | 由 `record_service` 统一分配，用于记录存储顺序 |
| `alarm_id` | 报警 ID | 由 `alarm_service` 统一分配，同一次报警从发生到结束使用同一个 ID |
| `event_action` | 事件动作 | `alarm_start`、`alarm_clear` 或 `alarm_instant` |
| `timestamp_ms` | 事件 tick 时间 | `alarm_start` 为发生时间，`alarm_clear` 为恢复时间 |
| `datetime` | 事件 RTC 时间 | `time_valid=1` 时有效 |
| `duration_ms` | 持续时间 | `alarm_clear` 事件填写，`alarm_start` 和 `alarm_instant` 为 0 |
| `alarm_reason` | 报警原因 | 当前规划支持 NTC 高温、NTC 低温和 TI1..TI5 开路 |
| `ntc_temp_milli_c` | 发生时 NTC 温度 | 单位 mC |
| `digital_input_state` | 事件发生时 TI raw 状态 | 底层 bit0..bit4 对应业务 TI1..TI5 |

展示示例：

```text
10:01:23 TI1 开路报警发生
10:05:41 TI1 开路报警恢复，持续 258s
10:08:12 浪涌峰值超过 5.0 kA
```

设计原则：

- 不在 `alarm_clear` 时回写 `alarm_start` 记录。
- 不在查询路径扫描 TSDB 配对同一个 `alarm_id`。
- UI、Modbus 和上位机都按事件日志顺序展示。
- 如果上位机需要闭合视图，可以自行按 `alarm_id` 离线合并。

## 当前活跃报警

当前活跃报警由 `alarm_service` 在 RAM 中维护，并在报警发生/结束边沿同步更新 EEPROM KVDB 中的活跃报警镜像。当前最大活跃报警数量固定按 6 个规划：TI1..TI5 各 1 个，NTC 1 个。

| 字段 | 含义 |
| --- | --- |
| `alarm_id` | 当前活跃报警 ID |
| `start_timestamp_ms` | 报警发生 tick 时间 |
| `start_datetime` | 报警发生 RTC 时间 |
| `duration_ms` | 当前已持续时间 |
| `reason` | 报警原因 |
| `source_index` | 报警源编号；NTC 为 0，TI1..TI5 为 1..5 |

活跃报警持久化原则：

- 报警发生时写一次 EEPROM KVDB，保存当前活跃报警镜像。
- 报警结束时写一次 EEPROM KVDB，移除对应活跃报警镜像。
- 不周期性写 `duration_ms`，持续时间由当前 tick/RTC 动态计算。
- EEPROM 镜像只用于复位恢复当前活跃报警，不作为历史日志查询入口。
- 完整报警历史仍写入片内 Flash TSDB。

当前活跃报警命令：

```text
alarm
```

输出示例：

```text
[INFO] alarm: active=1 total_started=3 total_ended=2
[INFO] alarm: id=3 reason=ti3_open source=3 start_ms=12000 duration_ms=2800
```

报警使能调试命令：

```text
alarm config
alarm enable ntc 0
alarm enable ti3 0
alarm enable ti3 1
```

这些命令只通过 `backend_request` 修改 `param_service` 参数，不直接改 `alarm_service` 内部状态。

## 当前支持的报警原因

| 枚举 | 字符串 | 含义 |
| --- | --- | --- |
| `RECORD_SERVICE_ALARM_REASON_NTC_HIGH` | `ntc_high` | NTC 温度高报警 |
| `RECORD_SERVICE_ALARM_REASON_NTC_LOW` | `ntc_low` | NTC 温度低报警 |
| `RECORD_SERVICE_ALARM_REASON_TI1_OPEN` | `ti1_open` | TI1 open 报警 |
| `RECORD_SERVICE_ALARM_REASON_TI2_OPEN` | `ti2_open` | TI2 open 报警 |
| `RECORD_SERVICE_ALARM_REASON_TI3_OPEN` | `ti3_open` | TI3 open 报警 |
| `RECORD_SERVICE_ALARM_REASON_TI4_OPEN` | `ti4_open` | TI4 open 报警 |
| `RECORD_SERVICE_ALARM_REASON_TI5_OPEN` | `ti5_open` | TI5 open 报警 |

## 当前默认判定条件

| 项目 | 触发条件 | 恢复条件 | 待确认 |
| --- | --- | --- | --- |
| NTC 高温 | `ntc_sensor_enable=1`、`ntc_high_alarm_enable=1` 且温度大于等于 `ntc_high_alarm_threshold_mc` 并持续超过 `ntc_alarm_debounce_ms` | 温度低于 `ntc_high_alarm_threshold_mc` 并持续超过 `ntc_alarm_debounce_ms`，或高温报警被禁用 | 不设置独立恢复回差 |
| NTC 低温 | `ntc_sensor_enable=1`、`ntc_low_alarm_enable=1` 且温度小于等于 `ntc_low_alarm_threshold_mc` 并持续超过 `ntc_alarm_debounce_ms` | 温度高于 `ntc_low_alarm_threshold_mc` 并持续超过 `ntc_alarm_debounce_ms`，或低温报警被禁用 | 不设置独立恢复回差 |
| TI 报警 | 默认低电平 open 触发，也可按用户配置 open 电平判断触发 | 默认高电平 close 恢复，也可按用户配置 close 电平判断恢复 | TI open/close 电平、报警使能、是否需要消抖 |

报警使能：

- `ntc_sensor_enable` 控制 NTC 传感器是否参与业务，产品默认值为 1。
- `ntc_high_alarm_enable` 独立控制 NTC 高温报警，产品默认值为 0。
- `ntc_low_alarm_enable` 独立控制 NTC 低温报警，产品默认值为 0。
- `ntc_alarm_debounce_ms` 控制 NTC 高温/低温报警发生和恢复消抖，产品默认值为 1000 ms。
- `ti_alarm_enable_mask` 独立控制 TI1..TI5，bit0..bit4 分别对应 TI1..TI5，产品默认值为 0x00。
- 浪涌报警默认开启，由 `surge_alarm_enable` 参数控制；浪涌详情参数另行设计。

## 当前实现边界

- `alarm_service` 只维护 NTC 和 DI 的活跃报警。
- 报警发生时更新 RAM 活跃表，并计划写 EEPROM KVDB 活跃报警镜像。
- 报警发生时生成一条 `alarm_start` 类型的 `RECORD_SERVICE_EVENT_ALARM` 记录，由 `record_wr` 异步写入 FlashDB TSDB。
- 报警恢复时生成一条 `alarm_clear` 类型的 `RECORD_SERVICE_EVENT_ALARM` 记录，由 `record_wr` 异步写入 FlashDB TSDB。
- 报警结束时计划从 EEPROM KVDB 活跃报警镜像中移除对应报警。
- TSDB 只追加事件，不回写旧记录，不保存对外可见的裸 Flash 地址。
- 报警历史记录保存在 `alarm_tsdb`，浪涌详情记录保存在 `surge_tsdb`，两者不是同一个存储集合。
- `record_service_query_alarm_latest_page()` 已支持按最新到最旧分页读取 `alarm_tsdb` 中的报警事件，并过滤开发测试记录。
- MSH 调试命令 `record alarms [offset] [count]` 已使用上述后端接口验证报警历史分页。
- Modbus 暂不冻结正式报警历史查询寄存器。
- 浪涌详情格式见 `浪涌报警事件记录格式.md`；浪涌高阈值报警使用 `alarm_instant` 事件动作。

## 待确认问题

- NTC 高温/低温报警阈值和消抖时间的出厂配置值。
- TI1..TI5 的具体客户侧业务含义和显示名称。
- 报警结束后是否需要记录确认人、确认时间或清除方式。
