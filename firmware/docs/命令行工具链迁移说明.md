# CMake/GCC/J-Link CLI 构建说明

本项目开始并行引入 GCC + CMake + Ninja + J-Link CLI 工具链。Keil 工程暂时保留作为回退路径，新的主线目标是让构建、烧录、调试流程都由文本脚本管理。

## 当前目标

- `surge_backend_serial`：后端优先串口测试固件。当前包含 RT-Thread、debug UART、MSH、data/param/record service、HDC1080、AT24C128 和现有 UI 编译依赖。

## 依赖工具

- CMake
- Ninja
- Arm GNU Toolchain，需提供 `arm-none-eabi-gcc`、`arm-none-eabi-objcopy`、`arm-none-eabi-size`
- SEGGER J-Link，当前优先使用 `C:\Program Files\SEGGER\JLink_V946\JLink.exe`

当前环境已检测到 CMake、Ninja、Arm GNU Toolchain 和 J-Link。`surge_backend_serial` 已可通过 GCC/CMake 完整生成 ELF、HEX、BIN 和 MAP。

## 构建命令

安装 Arm GNU Toolchain 并加入 PATH 后执行：

```powershell
.\tools\build_cmake.ps1
```

等价命令：

```powershell
cmake --preset gcc-debug -S .
cmake --build --preset gcc-debug --target surge_backend_serial
```

预期产物：

```text
build/gcc-debug/surge_backend_serial.elf
build/gcc-debug/surge_backend_serial.hex
build/gcc-debug/surge_backend_serial.bin
build/gcc-debug/surge_backend_serial.map
```

## J-Link CLI 烧录

生成 hex 后执行：

```powershell
.\tools\flash_jlink.ps1
```

默认脚本：

```text
tools/jlink/flash_backend_serial.jlink
```

脚本显式指定：

```text
device -AT32F403ARGT7
if SWD
speed 4000
```

本机旧版 `C:\Program Files\SEGGER\JLink\JLink.exe` 为 V7.58a，曾因设备名不完整或设备库差异弹出 Device Selection。当前脚本优先选择 V9.46，并使用精确设备名 `-AT32F403ARGT7`，对应 `AT32F403A_1024.FLM`。

不要直接运行未指定 `device` 的 `JLink.exe`，否则 J-Link Commander 可能弹出设备选择窗口。

## 移植说明

- GCC 构建使用 AT32 官方 `startup/gcc/startup_at32f403a_407.s`。
- GCC 构建使用 RT-Thread `libcpu/arm/cortex-m4/context_gcc.S`。
- 链接脚本为 `firmware/linker/AT32F403ARGT7.ld`，Flash 1 MB，RAM 96 KB。
- GCC 链接入口使用 `-Wl,-e,entry`，先进入 RT-Thread `rtthread_startup()`，再由 main 线程调用应用 `main()`。
- `board.c` 在 GCC 下使用 linker symbol `_end` 作为 RT-Thread heap 起点。

## 当前已知警告

- newlib/nosys 会提示 `_close`、`_fstat`、`_getpid`、`_isatty`、`_kill`、`_lseek`、`_read`、`_write` 未实现。当前没有阻塞链接，后续如果需要直接使用标准 `printf`/文件接口，再补完整 syscall stub。
- FinSH 的 MSH 导出宏在 GCC 下会产生函数指针类型转换警告，这是 RT-Thread/FinSH 现有宏风格导致，当前不影响固件产物生成。
