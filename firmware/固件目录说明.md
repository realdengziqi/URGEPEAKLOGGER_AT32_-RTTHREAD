# 固件目录说明

本目录存放参与固件构建的工程文件。

- `app`：应用入口和设备业务逻辑。
- `bsp/at32f403`：AT32F403ARGT7 板级支持和 RT-Thread 板级钩子。
- `external_port`：第三方中间件的项目适配层。
- `docs`：固件调试记录、硬件映射和设计说明。
- `project/mdk_v5`：Keil MDK 工程文件。
- `vendor`：已复制到固件工程内并参与构建的第三方或厂商源码。

## 当前状态

- AT32F403A_407 官方固件库 V2.2.2 已从 `external` 复制到 `vendor`。
- 目标 MCU 已确认为 AT32F403ARGT7。
- 早期 `AT32F403_Firmware_Library_V2.1.3` 厂商源码副本仅保留用于追溯，当前 Keil 工程不使用。
- RT-Thread 源码以 `external/rt-thread` 子模块形式跟踪，版本为 `v5.2.2`。
- RT-Thread v5.2.2 最小内核文件已复制到 `vendor/rt-thread-v5.2.2`，并已接入 Keil 工程。
- 当前硬件初始化覆盖已验证的调试串口、LED、按键、ST7789 SPI2 基础显示驱动、HDC1080 和 AT24C128C EEPROM。
- `drivers/sensor` 包含项目本地 HDC1080 I2C1 温湿度驱动。
- `drivers/storage` 包含项目本地 AT24C128C I2C2 EEPROM 驱动；参数持久化通过 FlashDB KVDB 使用这 16 KB EEPROM。
- `app_main` 当前处于后端优先模式。UI 自动初始化和 UI 按键轮询已暂停，使 COM12 串口测试优先聚焦后端数据访问。
- `app_data_service` 提供第一版后台数据服务线程和设备数据快照边界。当前线程名为 `data_svc`，优先级 18，栈 768 字节，更新周期 200 ms，并每 1000 ms 读取一次 HDC1080 环境温湿度。
- `app_param_service` 维护运行期参数表，执行范围校验，并通过 EEPROM KVDB 保存和加载参数镜像。
- `app_record_service` 提供运行期事件记录环形缓冲，当前容量 16 条，并将报警/普通事件与浪涌事件追加到片内 Flash TSDB。
- `app_ui` 保留为前端显示层，但在后端优先模式下不再自动启动。它只渲染 `data_service` 快照数据，不直接访问 ADC、Flash、Modbus 或硬件采样。
- `app_key_input` 扫描并消抖已验证的前面板按键，然后向 UI 层上报前端按键事件。
- `app_bringup` 封装 LED、按键和 LCD 的硬件 bring-up 命令，使 `app_main` 聚焦主循环和 CLI 分发。
- 已为后端串口目标新增并行 CMake/GCC 构建路径。Keil 在 GCC 构建、J-Link 烧录和 COM12 启动日志完全稳定前仍保留为备用路径。
